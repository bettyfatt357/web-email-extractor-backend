================================================================================
                    ARCHITECTURE HIGHLIGHTS
================================================================================

✓ Provider-Agnostic Authentication
  - Auth system designed to migrate to Clerk, Auth.js, Better Auth, Supabase
  - Only loadUserFromCredential() needs updating for new provider
  - Admin routes, components, hooks all stay unchanged

✓ Robust Error Handling
  - Google PSE: quota, rate limit, timeout handling
  - Queue: retry logic with exponential backoff
  - Extraction: JSDOM → Puppeteer fallback
  - Rate limiting: 429 responses wait 5s and retry

✓ Production-Ready Patterns
  - Server-side only credentials (ADMIN_CREDENTIAL never in browser)
  - Type-safe API client (no `any` types)
  - Centralized error handling
  - Comprehensive logging

✓ Scalability Design
  - Queue system ready for multiple workers
  - Configurable concurrency (WORKER_CONCURRENCY)
  - Configurable rate limiting (REQUEST_DELAY_MS)
  - Horizontal scaling ready (needs worker service)
